Homelab Blueprint

A single-node Proxmox layout for media streaming, optional NAS, and a separate books stack — documented with placeholders only.

Overview

This blueprint separates concerns across hypervisor guests so playback, automation, and reading stay independent. Application install details live in companion repositories; this page describes roles, storage, and how the pieces connect.

RoleResponsibility
HypervisorProxmox VE on the host — VMs, LXCs, storage pools
Media playerJellyfin (or similar) for streaming
Media automationRequest UI, Sonarr/Radarr, indexers, torrent client
NAS (optional)File shares and backup target
Books / readingSeparate guest with its own disk — Kavita + Calibre stack

Architecture

One physical host runs Proxmox. Guests share a dedicated media volume where hardlinks matter; the books stack keeps its library on an isolated disk.

Storage

Mount media volumes by LABEL or /dev/disk/by-id/... — never by unstable /dev/sdX letters.

DataWhereWhy
Hypervisor OSBoot deviceSmall, rebuildable
Guest virtual disksGuest storage poolSnapshots and backups
Movies / TV / downloadsDedicated media volumeLarge; shared by player + automation
Books / textbooksBooks guest diskIsolation from media server

Bind-mount the media volume into player and automation guests at the same absolute path (example: /data/media) so hardlinks work.

Network

Private LAN only unless you deliberately add a reverse proxy and TLS. Give each guest a static address you choose and document the map privately — not in public git.

Create

Condensed hypervisor setup — app installs are in the companion repos.

  1. Install Proxmox VE; prepare guest storage pool.
  2. Label the media disk; mount on host (example /mnt/media).
  3. Create media player, media automation, and books guests (optional NAS).
  4. Attach guests to LAN bridge; assign static IPs locally.
  5. Bind-mount media volume into player + automation at identical paths.
  6. Follow homelab-media-stack creation guide on player + automation guests.
  7. Follow homelab-books-stack creation guide on the books guest.
  8. Store credentials in a password manager — never in git.

Safety

  • No passwords, API keys, personal usernames, or private LAN addresses in public documentation.
  • Use placeholders (<PROXMOX_HOST>, etc.) and set real values only on your infrastructure.
  • Prefer LABEL/by-id disk mounts for removable media.
  • Keep books storage separate when you want media-drive isolation.
  • See SECURITY.md in the repo root for the redaction checklist.
